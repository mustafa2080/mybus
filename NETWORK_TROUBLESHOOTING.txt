# حل مشاكل الشبكة في Flutter/Android Build

## المشكلة الحالية:
- Read timed out عند تحميل dependencies
- مشاكل في تحميل mobile_scanner plugin
- NullPointerException في project evaluation

## الحلول المطبقة:

### 1. زيادة timeout values:
- connectionTimeout: 300 ثانية (5 دقائق)
- socketTimeout: 300 ثانية (5 دقائق)
- إعادة المحاولة: 10 مرات

### 2. إضافة repositories إضافية:
- JitPack.io كـ fallback
- Maven Central mirror
- تحسين Google repository

### 3. أوامر لحل المشاكل:

#### تنظيف شامل:
```bash
flutter clean
cd android
./gradlew clean
cd ..
flutter pub get
```

#### بناء مع إعدادات الشبكة:
```bash
flutter build apk --debug --verbose
```

#### إذا استمرت المشكلة، جرب:
```bash
cd android
./gradlew assembleDebug --refresh-dependencies
./gradlew assembleDebug --rerun-tasks
```

#### للمشاكل المستمرة:
```bash
# حذف cache
flutter pub cache clean
flutter clean
rm -rf ~/.gradle/caches/
# أو على Windows:
# rmdir /s %USERPROFILE%\.gradle\caches

# إعادة البناء
flutter pub get
flutter build apk --debug
```

### 4. نصائح إضافية:
- تأكد من اتصال إنترنت مستقر
- استخدم VPN إذا كان هناك حجب للمواقع
- جرب في وقت مختلف (قد تكون الخوادم مشغولة)
- تأكد من عدم وجود firewall يحجب التحميل

### 5. إعدادات proxy (إذا كنت تستخدم proxy):
أضف في android/gradle.properties:
```
systemProp.http.proxyHost=your.proxy.host
systemProp.http.proxyPort=8080
systemProp.https.proxyHost=your.proxy.host
systemProp.https.proxyPort=8080
```
