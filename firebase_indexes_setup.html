إعداد Firebase Indexes

حل مشكلة "The query requires an index" في نظام الشكاوى

المشكلة

الخطأ:

[cloud_firestore/failed-precondition] The query requires an index
السبب:

Firebase Firestore يتطلب إنشاء indexes مركبة عند استخدام عدة شروط مع orderBy في نفس الاستعلام.

الحل المطبق

تم حل المشكلة بطريقتين:

  1. تبسيط الاستعلامات: إزالة orderBy من الاستعلامات وترتيب النتائج محلياً
  2. إنشاء Indexes: إنشاء الفهارس المطلوبة في Firebase Console

1 الحل السريع (مطبق بالفعل)

تم تعديل الكود لتجنب الحاجة للـ indexes المعقدة:

قبل التعديل (يسبب خطأ):
// ❌ يتطلب compound index return _firestore .collection('complaints') .where('parentId', isEqualTo: parentId) .where('isActive', isEqualTo: true) .orderBy('createdAt', descending: true) // ❌ مشكلة هنا .snapshots();
بعد التعديل (يعمل بنجاح):
// ✅ لا يتطلب compound index return _firestore .collection('complaints') .where('parentId', isEqualTo: parentId) .where('isActive', isEqualTo: true) .snapshots() .map((snapshot) { final complaints = snapshot.docs .map((doc) => ComplaintModel.fromMap(doc.data())) .toList(); // ✅ ترتيب محلي بدلاً من orderBy complaints.sort((a, b) => b.createdAt.compareTo(a.createdAt)); return complaints; });

2 إنشاء Indexes يدوياً (اختياري)

إذا كنت تريد استخدام orderBy في الاستعلامات، يمكنك إنشاء الفهارس التالية:

Index للشكاوى حسب ولي الأمر:
  • Collection: complaints
  • Field 1: parentId (Ascending)
  • Field 2: isActive (Ascending)
  • Field 3: createdAt (Descending)
Index للشكاوى حسب الحالة:
  • Collection: complaints
  • Field 1: status (Ascending)
  • Field 2: isActive (Ascending)
  • Field 3: createdAt (Descending)
ملاحظة مهمة

الحل الحالي يعمل بدون الحاجة لإنشاء هذه الفهارس. الترتيب يتم محلياً وهو سريع للبيانات الصغيرة والمتوسطة.

ملاحظات الأداء

المزايا
  • لا يتطلب إنشاء indexes
  • يعمل فوراً بدون انتظار
  • مرونة في التعديل
  • أقل تعقيداً
اعتبارات
  • الترتيب يتم على الجهاز
  • مناسب للبيانات المتوسطة
  • قد يحتاج indexes للبيانات الكبيرة
  • استهلاك bandwidth أكثر قليلاً

حلول بديلة

تقسيم النتائج إلى صفحات صغيرة:

  • جلب 20 شكوى في كل مرة
  • استخدام startAfter للصفحة التالية
  • تحسين الأداء للبيانات الكبيرة

معالجة البيانات على الخادم:

  • إنشاء Cloud Function للاستعلامات المعقدة
  • ترتيب وتصفية على الخادم
  • إرجاع النتائج جاهزة

إعادة تصميم هيكل البيانات:

  • فصل الشكاوى حسب الحالة في collections منفصلة
  • استخدام subcollections
  • تقليل الحاجة للاستعلامات المعقدة
